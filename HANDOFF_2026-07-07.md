## 🔴 最優先タスク: 建設停滞の解消(修正はコミット済み・未デプロイ)

**症状**: 建設待ち64件が数時間まったく減らない。
**根因(診断済み)**: 辺境の7つのbuilder hutのbuilderたちの**自宅が中心部**にあり、
10倍速では1日=実2分なので、通勤(180ブロック超)だけで日中が終わる。
資材・チャンクロード・健康は全て確認済みで問題なし。job AIはSTART_WORKINGのまま。

**修正は実装済み**: `/assignHome?x&y&z&citizenId`(市民の自宅を指定住居に引っ越し)
— commit `bridge: /assignHome endpoint` に含まれる。**まだデプロイしていない**。

**手順(機械的に実行できる)**:
1. skill `deploy-bridge` の手順どおり: ビルド→stop→rebuild_bridge→start --no-agents
   →tickrate 10x→supply_bot/council起動確認(CLAUDE.mdのサブシェルcd起動を厳守)
2. 7つの辺境builder hutとそのbuilder(市民ID)を確認:
   `curl -s http://localhost:8089/status` で `blockhutbuilder` の workers を見る。
   ハット位置: (264,25) (57,174) (114,366) (315,336) (387,159) (18,261) (96,65)
3. 各builderについて、**そのハットから60ブロック以内にある建設済み(level≥1)の
   blockhutcitizen** を status から探し、
   `curl -X POST '/assignHome?x=<住居x>&y=<住居y>&z=<住居z>&citizenId=<builderのID>'`
   で引っ越しさせる(満室なら次に近い住居を試す。応答が residence full)
4. 検証: 30〜60分放置して `debugWorkOrders` の件数が減り始めればOK。
   減らなければ `debugBuildGates` と colony-diag skill で再診断
