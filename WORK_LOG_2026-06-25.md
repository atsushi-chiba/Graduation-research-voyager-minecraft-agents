# 作業ログ 2026-06-25

## 概要
2026-06-24の続き。「MineColoniesのスキル・社会システムを持ったLLMエージェントによる自律的な街づくり」というアイデアを検証した。当初は「mineflayerでMineColoniesサーバーに接続して同じことができないか」を試したが、技術的な壁にぶつかり、最終的に**自作のForge Modでサーバー内部から直接操作するHTTP Bridge方式**に切り替えて、実際にLLMが配置した建物をNPCが建築し始めるところまで確認できた。

## やったこと

### 1. MineColoniesサーバーの構築
- Forge 1.20.1-47.1.3 + MineColonies + Structurize + BlockUI + Domum Ornamentum + MultiPiston を `/root/mc-server-forge`(ポート25566)に構築。元のバニラサーバー(`/root/mc-server`、25565)とは別。
- CurseForgeのAPIはこのサーバーのIPをブロックしていたため、Mod本体はユーザーが手動ダウンロード→Tailscale経由でscp転送。
- ワールドはスーパーフラットに設定。

### 2. mineflayerでは接続不可と判明
- MineColonies/Structurizeが必須のFML通信チャンネルを登録しているため、バニラプロトコルのbot(mineflayer)は**ログイン自体を拒否される**(`This server has mods that require Forge to be installed on the client.`)。
- 建物配置やコロニー設立もGUI操作前提で、コマンドだけでは完結しない。

### 3. Bridge Mod(`voyagerbridge`)を自作
場所: `voyager/env/minecolonies-bridge/`(`minecolonies-experiment`ブランチ)

- サーバーと同じJVM内で動くForge Modとして実装し、HTTP API(ポート8089)を生やすことでクライアント接続を完全にバイパス。
- MineColoniesの自作クラス(`IColonyManager`等)はForgeのobfuscation対象外(バニラのみSRG化される)と分かったため、`fg.deobf(files(...))`が不完全でも**読みやすいメソッド名で直接呼び出せる**ことを確認。
- 実装したエンドポイント:
  - `POST /place` — hutブロックを座標指定で設置(blueprint自動割り当て込み。`BLUEPRINT_PATHS`という型名→ファイルパスのマップを、minecolonies.jar内の実ファイルから調査して構築)
  - `POST /found` — コロニーを設立(`CreateColonyMessage`相当)
  - `POST /spawnCitizen` — 市民を増やす
  - `GET /status` — コロニー一覧(id/名前/座標/市民数)
  - `POST /requestBuild` — 建築指示(work order)を出す。**これが無いと配置しただけでは永遠に着工しないことが判明**(`BuildRequestMessage`相当)
  - `POST /giveToCitizen` — 市民の個人インベントリに直接資材を挿入(倉庫がレベル0でまだ機能しないため、当面の代替手段)

ビルドはJDK17必須(Forge MDK付属のGradle 8.1.1がJDK21で動かない)。`/opt/jdk-17.0.19+10` に導入済み。

### 4. mayor_agent.js
- LLM(OpenRouter経由)が`/status`を見て次の一手(配置/コロニー設立/市民追加)をJSONで判断し、Bridge APIを叩くだけのシンプルなループ。
- 実行確認済み: LLMが`blockhutbuilder`等を自律的に選んで配置し、blueprintも正しく割り当てられることを確認。

### 5. 実際に建築家NPCが動くまでの泥臭い調整
ここが今日の本番。配置だけでは何も起きず、以下を順番に解決した:

1. **work orderが無い** → `/requestBuild`を追加して解決
2. **Builder's Hutがレベル0だと他の建物の仕事を受けられない** → まずBuilder's Hut自身を建てる必要があると判明
3. **倉庫もレベル0で機能しない** → 倉庫経由の資材搬入ができないので、`/giveToCitizen`で建築家(Andre)に直接資材を入れる方式に
4. **必要資材の特定** → Wiki頼みではなく、実際の`builder1.blueprint`ファイル(NBT)を解析して正確な個数を算出(オーク原木80、階段65、ハーフブロック35、丸石30、丸石階段30、砂利23、壁掛けトーチ11、ドア8、板材7、フェンスゲート7、フェンス2、他細かいもの数個)
5. **ツールのtier制限** → 鉄ツールを渡したが「建築家のレベルが低いと使えない」ことが判明。**MineColoniesの市民は自分のレベル以上のtierの道具を使えない**(石/木製に変更して解決)。この知見はメモリに記録済み([[minecolonies-tool-tier-gating]])
6. **Domum Ornamentumの「Framed」装飾ブロック** → 表示名が分かりにくいが、`stripped_oak_log`と`stripped_oak_wood`(原木と木は別アイテム)が必要だった
7. **夜になると作業しない** → `/time set day` + `gamerule doDaylightCycle false`で常時昼に固定

最終的に**Andreが資材とツールを使ってBuilder's Hutの建築作業を開始**したことを確認(セッション終了時点)。
