### ★通常ワールドでの founding の掟(ハマった落とし穴)
- **founding 前に townhall チャンクを forceload 必須**。遠方/無人サイトは未ロードで tile entity が null になり
  `/found` が NullPointerException(getTileEntity null)。対処: 起動を `FORCELOAD_CX=501 FORCELOAD_CZ=-319 FORCELOAD_R=80
  bash start_server.sh --no-agents` で行い中心を先にロード → `/surfaceY` で真の地表Y取得 → その Y で `/place`+`/found`。
- **設置Yはアンカー列の実地表(OCEAN_FLOOR)ちょうど**。1ずれると埋没/浮きで found が壊れる。footprint の mode Y ではなくアンカー列のY。
- 壊れた半founding コロニーは `colony delete` がクリック確認必須で消せない → seed同一なので **rm world して再生成**が最速リセット。

### ★サーバー/デーモン再起動の手順(座標依存)
- **サーバー再起動**: forceload が既定(200,200)に戻るので必ず `FORCELOAD_CX=501 FORCELOAD_CZ=-319 FORCELOAD_R=80` を付けて起動。
  起動後 `curl -s -X POST 'http://localhost:8089/tickrate?auto=true'`。
- **council 再起動**: anchor は colony 中心から**動的取得**に修正済み(ca066f5)なので ANCHOR env 不要。colony_watch の env なし再起動でも正しい。
- **供給デーモン**: `(cd .../minecolonies-bridge && setsid nohup node supply_bot.js >> supply_bot.log 2>&1 &)`。council も同様(env不要)。

### 残タスク/既知の軽微問題(次セッション)
- **Agent C(terraform)未実装**: 平坦さゲート不合格時は「最も平坦な乾燥候補」にフォールバックし `TERRAFORM NEEDED` を warn。
  緩斜面は builder の自前整地に依存。**builder整地の合格ライン実測 → `VOYAGER_PLACENEXT_FLATNESS` の T を確定**(設計 D2)。
  - **★実地の知見(2026-07-11, 建物38時点)**: 乾いた平坦コアは最初の約38建物で埋まり、その先の territory は水・丘が多い。
    placeNext が `no dry buildable site ... spread 22 ... TERRAFORM NEEDED` を頻発し急斜面へフォールバック。builder hut 増設は
    `no valid position found within territory` で失敗(territoryが建物で飽和)。=> 次の優先度: (1) Agent C(terraform)で急斜面を平坦化、
    または (2) **flatness T を緩斜面向けに引き上げ**(VOYAGER_PLACENEXT_FLATNESS=8〜10、要サーバー再起動)て builder自前整地に寄せる、
    (3) 市長の placeNext 乱立を抑える/MAX_DIST 調整で territory を使い切らせない。builder は現状1人=建設律速(複数化は operator 役割だが territory 空き待ち)。
- サーバー起動の FORCELOAD 座標は未永続(手動env)。start_server.sh に既定を焼くか env ファイル化を検討。
- 市長の建物乱立(governorで抑制されるが多め)、`blockhutforester` 幻覚(一度きり自己修正・軽微)、blacksmith等は University 研究待ちで発注拒否(正常進行)。

  各機能は狭いスコープでコード生成のみ、**ライブ系に触らせない**。あなたが受け取り直列デプロイ。

### 機能バックログ(優先順)
1. **新サーバー/world構築(先行・直列で自分でやる)**: server.properties の level-type を非フラットへ、
   新worldディレクトリ、spawnバイオーム plains 固定。旧worldは `world_backup_<timestamp>` で退避。

   地表ハイトマップをサンプリング。既存の衝突判定/builder半径100/groundlevelタグY補正は流用。
3. **bridge terraform(フォールバック)**: 急斜面のみ setblock で平坦化。**まず builder の自前整地
   (blueprint範囲の掘削/埋立)で足りるか実測**してから作る。
4. **チート供給の縮退**: supply_bot を安全網化。miner/lumberjack/farmer の生産を確認。demand.json 連動。
